## Rolling-daily-cap enforcement closure - 2026-06-11

- Completeness audit found the one genuine implementation gap in the repo: `rolling_daily_cap_mist` was stored end-to-end (Move `Policy`, `set_limits`, on-chain fetch in `enclave/src/policy_source.rs`, env/Docker build-arg plumbing) but never enforced — `evaluate_policy` only checked recipient/package/per-tx/bps, so a drainer holding the passkey could drip many under-per-tx-cap transactions past the daily cap.
- Fix: added `enclave/src/ledger.rs` (`SpendLedger`) — a per-vault rolling 24h window of co-sign-approved outflows, keyed by lowercased vault address, digest-deduplicated so re-signing the same transaction never double-counts, pruned per request, conservative on backwards clock steps. `evaluate_policy_with_window_outflow` rejects when prior-window outflow plus the requested net outflow exceeds the cap (reason `rolling daily outflow exceeds policy cap`); `co_sign_transaction` consults the ledger before signing and records the outflow only after a signature is issued (refusals consume nothing). `policy_check_and_cosign` remains the stateless single-tx preview. The cap stays opt-in (0 = disabled), so existing deployments and refusal reasons are unchanged.
- Evidence: `CARGO_HOME=/private/tmp/aegis-cargo cargo test` in `enclave` passed 23 tests (was 14) including `refuses_drip_drain_that_exceeds_rolling_daily_cap_across_transactions` and 5 `SpendLedger` tests. `pnpm test:integration:enclave-cosign` now also proves the stateful path over real HTTP with `AEGIS_ROLLING_DAILY_CAP_MIST=1500000000`: benign 1-SUI co-sign signed, same-digest retry signed (`sameDigestRetrySigned: true`), second 1-SUI digest refused with `rollingDailyCapRefusalReason: "rolling daily outflow exceeds policy cap"`. `pnpm test:integration:vault-execute` re-passed live on 2026-06-11: benign passkey+local-enclave 2-of-2 testnet digest `J42Y7dr1mAPfdzLpTuccVsxgGfUHmHPVKHg9Znrxyhv9` with the seeded drain still refused (`recipient is not allowlisted`). `pnpm typecheck`, `pnpm lint`, and `./app/node_modules/.bin/biome check scripts/enclave-cosign-smoke.ts` passed.
- Honest scope note: the ledger is enclave-memory state; an enclave restart clears the window (it does not weaken any per-tx/bps/allowlist check). On-chain rolling-cap accounting stays out of scope because vault transactions are plain PTBs that never touch `aegis::policy` — the co-signer is the enforcement point by design.
